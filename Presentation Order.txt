Title:
Increasing GPU Computation Efficiency by Overlapping Memory Data Transfer and Kernel Computation on Device

Overview:
In this presentation, I will explain why overlapping memory transfer and kernel computation is useful, introduce CUDA streams, explain how non-default streams make overlap possible, walk through example code, and compare performance results.

Introduction:
A basic GPU program usually performs:
- Data transfer from CPU to GPU
- Kernel computation on the GPU
- Result transfer from GPU back to CPU

The problem is that during memory transfers, the CPU or GPU may sit idle. CUDA streams help solve this by splitting data into chunks, so while one chunk is being transferred, another chunk can be computed.

Conclusion:
- CUDA streams allow independent operations to run concurrently
- cudaMemcpyAsync and pinned memory allow memory transfers to overlap with kernel execution
- Overlapping transfer and computation can reduce GPU idle time
- This improves throughput in high-performance computing applications
